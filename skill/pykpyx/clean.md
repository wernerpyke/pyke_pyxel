# /pykpyx clean

You are executing the `clean` sub-command. This deletes all generated game code.

## Arguments

- `[path/to/spec.md]` (optional) — path to the spec file whose generated code should be cleaned.
- `[target-dir]` (optional) — the game code directory to delete. Default: derived from the spec **filename** (lowercase, snake_case, e.g. `MY-GAME.md` → `my_game/`).

If no spec path or target directory is provided, use the same auto-discovery logic as the validate command (look in `specs/`, excluding `*-VALIDATION.md`). If exactly one spec is found, derive the target directory from its filename.

## Workflow

### 1. Determine target directory

Resolve the game code directory to delete:

- If `[target-dir]` is provided, use it directly.
- If `[path/to/spec.md]` is provided, derive the directory from the spec filename.
- Otherwise, auto-discover the spec and derive the directory from its filename.

### 2. Check the directory exists

If the target directory does not exist, report: *"Nothing to clean — `<target-dir>/` does not exist."* and stop.

### 3. Confirm with the human

**Always ask for confirmation before deleting.** List the files that will be deleted and ask:

> The following files will be permanently deleted:
>
> - `<target-dir>/main.py`
> - `<target-dir>/game_loop.py`
> - `<target-dir>/game_load.py`
> - `<target-dir>/__init__.py`
> - *(any other .py files found)*
>
> This will also remove the `<target-dir>/` directory if it is empty after deletion.
>
> Proceed? (yes/no)

Only proceed if the human confirms.

### 4. Delete generated files

Delete all `.py` files in the target directory. These are the files generated by the `generate` command.

**Do not delete:**
- The `specs/` directory or any spec files
- The `assets/` directory or `.pyxres` files
- `scripts/run_game.sh`
- `pyproject.toml`
- Any files outside the target directory

### 5. Remove the directory

If the target directory is empty after deleting the `.py` files, remove it.

If non-`.py` files remain (e.g. the human added data files), warn: *"`<target-dir>/` still contains non-generated files and was not removed."*

### 6. Report

```
Cleaned game code from <target-dir>/:

  Deleted: <n> files
  Directory removed: yes | no (non-generated files remain)

To regenerate, run:
  /pykpyx generate specs/<SPEC-NAME>.md
```
